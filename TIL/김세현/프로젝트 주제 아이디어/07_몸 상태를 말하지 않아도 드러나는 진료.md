## 📌 프로젝트 개요

- **대주제**: Web 기술
- **핵심 기술**: WebRTC
- **플랫폼**: 모바일 웹 / PC 웹
- **타겟**:
    - 노인
    - 아동
    - 외국인
    - 의사소통이 어려운 환자
    - 증상 설명이 힘든 사람
- **목표**:
    - 환자가 증상을 ‘설명’하지 않아도
        
        **의사가 먼저 상태를 관찰하고 진료를 시작**할 수 있는 원격 진료 보조 플랫폼
        
    - 의료 행위가 아닌 **진료 이전의 상태 관찰을 돕는 보조 플랫폼**!!

---

## 🎯 기획 배경

### 왜 이 서비스를 만들었는가?

- 진료는 대부분 **말을 잘하는 사람에게 유리**
- 실제로 많은 환자들이:
    - 어디가 아픈지 정확히 모르고
    - 언제부터 아픈지도 헷갈리고
    - 설명하다가 중요한 걸 놓침

특히:

- 노인: 표현력 저하
- 아동: 증상 인식 부족
- 외국인: 언어 장벽
- 정신적·신체적 약자: 말 자체가 부담

⇒  **“진료의 시작은 설명이 아니라 관찰이어야 한다”**

---

## 💡 핵심 아이디어

> **진료를 ‘대화’로 시작하지 않고 ‘관찰’로 시작한다**
> 
- 환자는 말하지 않아도 됨
- 의사는 WebRTC 영상으로
    - 얼굴색
    - 호흡
    - 움직임
    - 반응 속도
        
        를 먼저 확인
        
- 말은 **두 번째 단계**

## ⚙️ 주요 기능

### ✅ 무언 사전 진료 모드 (Silent Pre-Check)

- 진료 시작 후 일정 시간:
    - 말하기 **X**
    - 질문 **X**
- 오직 관찰만 진행
    - 얼굴 창백/홍조
    - 호흡 속도
    - 눈 깜빡임 빈도
    - 자세 불안정 등…

---

### ✅ 상태 체크 가이드 UI (의사용)

- 의사 화면에 체크리스트 제공
    - 호흡 안정
    - 얼굴색 정상
    - 움직임 원활

📌 **의사의 관찰을 구조화**

---

### ✅ 최소 응답 유도 시스템

- 긴 설명 **X**
- 단답 선택 **O**

예:

- “아프면 고개를 끄덕이세요”
- “불편하면 손을 들어주세요”

➡️ **몸짓 기반 소통**

1. 환자 웹 접속
2. 카메라 허용

4. 의사 관찰

## ❓ Q&A

### Q1. 왜 앱이 아니라 웹인가요?

**A.** 

이 서비스의 주요 대상은 디지털 접근성이 낮은 사용자입니다. 
앱 설치는 큰 진입 장벽이 될 수 있기 때문에

**링크 클릭만으로 접속 가능한 웹**이 더 적합하다고 판단했습니다.

### Q2. 컴퓨터가 없는 노인이나 취약계층은 어떻게 사용하나요?

**A.** 

이 서비스는 **PC가 필요 없습니다.**

- 모바일 웹 기반
- 스마트폰만 있으면 사용 가능
- 앱 설치 불필요
