### 🚨 위험을 대신 봐주는 눈

> “위험한 순간, 혼자가 아니라 여러 명의 눈과 함께 걷는 서비스”
> 

### 📌 프로젝트 개요

- **대주제**: Web 기술
- **핵심 기술**: WebRTC
- **플랫폼**: 모바일 웹 / PC 웹
- **목표**:
    
    위험 상황에서 사용자가 직접 신고하지 않아도
    
    **제3자의 시야와 판단으로 위험을 조기에 감지**하는 안전 플랫폼
    

---

### 🎯 기획 배경

**왜 이 서비스를 만들었는가?**

- 위험 상황은 **확신되기 전까지 아무 행동도 못 함**
- 신고 버튼은:
    - 늦거나
    - 누르기 어렵거나
    - 상황 인지가 불가능한 경우가 많음
- 대부분의 안전 서비스는 **사후 대응 중심**

➡️ **사람이 느끼는 ‘쎄함’을 기술로 연결**하고자 함

---

### ❗ 문제 정의 (Problem Definition)

**실제로 자주 발생하는 상황**

- 밤길 귀가
- 배달 / 야간 근무
- 혼자 있는 공간 (원룸, 주차장)
- 낯선 사람과의 근접 상황

**공통 문제점**

- 위험을 느껴도 **확신이 없어서 행동 불가**
- 도움 요청 시점이 늦어짐
- 혼자 판단 → 대응 실패

---

### 👥 서비스 구성

**1️⃣ 사용자 (위험 당사자)**

- 모바일 웹 접속
- 카메라 ON
- 실시간 영상 송출
- 최소한의 조작

**2️⃣ 관찰자 (Guardian)**

- 가족 / 친구 / 자원봉사자
- 관찰자 전용 화면
- 말 없이 상황 모니터링
- 위험 감지 시 버튼 클릭

---

### ⚙️ 주요 기능

**✅ 공동 시야 시작**

- “같이 봐줘” 버튼 1회 클릭
- 자동으로:
    - 카메라 활성화
    - 관찰자 연결 요청

---

**✅ 관찰자 모드 (Guardian View)**

- 실시간 영상 시청
- 위치·시간 정보 표시
- 개입 없이 관찰 중심

---

**✅ 위험 감지 알림**

- 관찰자가 위험 감지 버튼 클릭
- 사용자에게 진동/알림 전송
- 추가 관찰자 자동 연결

---

**✅ 원버튼 신고 시스템 (Emergency One-Tap)**

- 위험 신호가 **여러 번 누적될 경우** 자동으로 활성화되는 신고 기능
- 사용자가 직접 상황을 판단하지 않아도
    
    **관찰자들의 판단을 기반으로 ‘신고 가능 상태’에 진입**
    

---

**✅ 다자 시야 확장**

- 위험 감지 시
- 관찰자 수 자동 증가
- 혼자 대응하지 않는 구조

---

**✅ 상황 종료 및 기록**

- 사용자가 안전 도착 시 세션 종료
- 자동 로그 생성:
    - 시간
    - 위치
    - 위험 발생 시점
