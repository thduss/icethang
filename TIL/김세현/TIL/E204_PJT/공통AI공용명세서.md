## 가상환경 만드는 코드

| 명령어                              | 의미               |
| -------------------------------- | ---------------- |
| `conda create -n 이름 python=3.10` | **빈 가상환경 생성**    |
| `conda env create -f xxx.yaml`   | **설계도 기반 환경 생성** |
| `conda activate 이름`              | 환경 사용 시작(가상환경 활성화)        |
| `conda deactivate`               | 환경 나가기           |

## Conda 기본 가상환경

### base란?
- Conda(아나콘다/미니콘다)의 기본 가상환경
- Conda 설치하면 자동으로 딸려오는 환경
- Python + 패키지들이 이미 들어 있음

    👉 그래서 (base)가 보이면 **“Conda 가상환경이 활성화된 상태”**


### 📊 venv vs conda(base)

| 구분         | venv             | conda(base)               |
| ---------- | ---------------- | ------------------------- |
| 정체         | Python 기본 가상환경   | Conda 가상환경                |
| 생성 도구      | `python -m venv` | `conda create`            |
| Python만 관리 | ⭕                | ❌ (Python + 라이브러리 + C패키지) |
| 가벼움        | ⭕ 매우 가벼움         | ❌ 상대적으로 무거움               |
| 웹/Django   | ⭐⭐⭐⭐             | ⭐⭐                        |
| 데이터분석/ML   | ⭐⭐               | ⭐⭐⭐⭐⭐                     |
| 프로젝트 단위    | 폴더 안에 생성         | 전역 위치에 생성                 |


- **개념은 같다** -> “환경을 분리해서 패키지 충돌 막자”
- **도구와 철학이 다르다**


### ◾ 쉽게 비유하면
- `venv` -> 도시락
    - 필요한 것만 담아서 프로젝트별로 들고 다님

- `conda base` -> 회사 구내식당
    - 기본 세팅 다 돼 있음, 무겁지만 편함
